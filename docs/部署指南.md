# 部署指南（上架步驟）

本指南帶你完成：① 申請金鑰 → ② 建立資料庫 → ③ 推上 GitHub → ④ 部署到 Vercel。
全程使用免費資源；Google Maps 在低流量下也落在免費額度內。

---

## 步驟 ① 申請 Google Maps API Key

1. 前往 <https://console.cloud.google.com>，登入並建立專案（例如 `twsmokemap`）。
2. **計費 Billing → 建立帳單帳戶**，綁定信用卡（必要；低流量不會扣款）。
3. **API 和服務 → 程式庫 → 啟用「Maps JavaScript API」**。
4. **憑證 → 建立憑證 → API 金鑰**，複製金鑰。
5. **保護金鑰（重要）**：
   - 點該金鑰 → 應用程式限制 → **HTTP 參照網址**，加入：
     - `http://localhost:5173/*`
     - `https://你的vercel網址/*`（部署後補上，例如 `https://twsmokemap.vercel.app/*`）
   - API 限制 → 僅勾選「Maps JavaScript API」。
6. **設預算警示防爆量**：計費 → 預算與快訊 → 新增預算（例如月上限 NT$300）＋ Email 通知。

## 步驟 ② 建立 Supabase 資料庫

1. 前往 <https://supabase.com>，建立免費專案（記住資料庫密碼）。
2. 左側 **SQL Editor**，貼上並執行 [`supabase/schema.sql`](../supabase/schema.sql)（建立 PostGIS、資料表、RPC）。
3. 再執行 [`supabase/policies.sql`](../supabase/policies.sql)（設定 RLS 權限）。
4. **Project Settings → API**，記下：
   - `Project URL`
   - `anon public` key
   - `service_role` key（僅供匯入腳本，切勿外洩）

### 匯入台北市吸菸區資料

1. 在專案根目錄建立 `.env`（複製 `.env.example`），填入：
   ```
   VITE_GOOGLE_MAPS_API_KEY=你的Google金鑰
   VITE_SUPABASE_URL=你的ProjectURL
   VITE_SUPABASE_ANON_KEY=你的anon金鑰
   SUPABASE_SERVICE_ROLE_KEY=你的service_role金鑰
   ```
2. 執行匯入：
   ```bash
   npm run import:taipei
   ```
   完成後 Supabase 的 `smoking_areas` 會有上百筆台北市官方吸菸區。

## 步驟 ③ 推上 GitHub

1. 在 GitHub 建立一個新的空 repo（例如 `twsmokemap`，不要勾 README）。
2. 在專案根目錄執行（把網址換成你的）：
   ```bash
   git remote add origin https://github.com/你的帳號/twsmokemap.git
   git push -u origin main
   ```
   > `.env` 已被 `.gitignore` 忽略，金鑰不會上傳，安心。

---

## 步驟 ④ 部署到 Vercel

1. 前往 <https://vercel.com>，用 GitHub 帳號登入。
2. **Add New → Project → Import** 你的 `twsmokemap` repo。
3. Framework 會自動偵測為 Vite（已附 `vercel.json`），直接沿用預設即可。
4. **Environment Variables** 加入（與 `.env` 相同，但**不要**加 service_role）：
   - `VITE_GOOGLE_MAPS_API_KEY`
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
5. **Deploy**。完成後會得到網址（如 `https://twsmokemap.vercel.app`）。
6. 回到步驟 ① 第 5 點，把這個正式網址加進 Google 金鑰的 HTTP 參照網址限制。

---

## 步驟 ⑤ 部署後驗證

- 用手機與電腦各開一次網址，走完三大功能。
- 跑效能檢測：
  ```bash
  npx lighthouse https://你的網址 --view
  ```
  確認 Performance / Accessibility ≥ 90，並把結果填入 [測試報告.md](測試報告.md)。
- 到 Google Cloud 確認 API 用量在免費額度內、預算警示已啟用。
